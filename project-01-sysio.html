syshr.io: SaaS
recherche d’entreprises & candidatures auto

syshr.io permet de trouver des entreprises tech près de chez soi et d’envoyer automatiquement sa candidature selon ses critères (poste, stack, localisation).

React Symfony Stripe MySQL Docker Nginx GitHub Actions

Fiche projet

Objectif

Centraliser la recherche d’entreprises tech locales + automatiser les candidatures.

Mon rôle

Développement fullstack : front React, API Symfony, base MySQL, déploiement Docker.

Points forts

Automatisation, parcours utilisateur clair, paiement Stripe, pipeline CI/CD.

Stack

Front: React
Back: Symfony (API) + Doctrine
Paiement: Stripe
DB: MySQL
Infra: Docker + Nginx
CI/CD: GitHub Actions

Fonctionnalités

  • Recherche d’entreprises tech par proximité (zone/rayon)
  • Filtres (poste, technos, type d’entreprise) + tri des résultats
  • Profil candidat : infos, CV, préférences, message de candidature
  • Envoi automatique de candidatures (selon critères définis)
  • Historique & suivi : candidatures envoyées, statut, relances (si activé)
  • Abonnement / paiement via Stripe

Points techniques

  • API Symfony structurée (routes, services) + Doctrine (entités/migrations)
  • Sécurité : authentification + contrôle d’accès (rôles/permissions)
  • Automatisation : jobs/cron pour déclencher les envois à intervalle
  • Stripe : gestion checkout / webhooks (abonnements / paiement)
  • Docker : environnement reproductible (front/back/db) + Nginx reverse proxy
  • CI/CD GitHub Actions : build/tests + déploiement automatisé

Ce que j’ai appris

Concevoir un SaaS “from scratch” : API + front, industrialiser le déploiement (Docker + CI/CD), intégrer Stripe proprement (webhooks) et fiabiliser l’automatisation d’envoi (jobs).
