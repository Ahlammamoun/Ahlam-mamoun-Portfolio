Cinephora
App cinéma (React + API)

Application web dédiée au cinéma : catalogue de films, détails, recherche & filtres et parcours utilisateur (compte/connexion), avec un backend API.

🎬 Voir le site ← Retour aux projets
React Symfony MySQL Docker Nginx GitHub Actions

Fiche projet

Type

Application web cinéma : navigation, recherche, pages film et séances.

Mon rôle

Développement fullstack : front React, API Symfony, base MySQL, conteneurisation Docker + Nginx et CI/CD.

Focus

UX fluide : navigation rapide, filtres utiles et pages détaillées propres.

Stack

Front: React
Back: Symfony (API) + Doctrine
DB: MySQL
Infra: Docker + Nginx
CI/CD: GitHub Actions

Fonctionnalités

  • Catalogue de films + pages détaillées (synopsis, infos)
  • Recherche et filtres (catégories, séances, etc.)
  • Connexion / compte utilisateur (si activé)
  • Espace admin : gestion des films / catégories / séances (si activé)
  • Affichage dynamique des contenus via API

Points techniques

  • Routing + structure composants React (pages, layout, composants UI)
  • Consommation API : fetch/axios, gestion erreurs, états de chargement
  • API Symfony : endpoints, validation, sécurité (rôles/permissions si admin)
  • MySQL + migrations Doctrine pour les données films/séances
  • Docker + Nginx : environnement reproductible et déploiement stable
  • CI/CD GitHub Actions : build/tests + déploiement automatisé

Ce que j’ai appris

Construire une app orientée données avec une UI réactive, organiser un front React propre (composants/routing), et fiabiliser l’ensemble via une API Symfony + Docker + CI/CD.
